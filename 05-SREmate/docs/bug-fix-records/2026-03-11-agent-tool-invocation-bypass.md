# Bug 记录：Agent 跳过工具调用直接生成 JSON

**发现日期**：2026-03-11

---

## 问题描述

用户输入 `826031111000001859合同数据` 时，Agent 没有调用任何工具，直接输出了 JSON 数据。

**与之前 Bug 的区别**：
- 之前是"调用多个工具"，这次是"不调用工具"

**根因**：
1. **对话历史导致 LLM 模仿模式**：用户之前查询过报价单，对话历史中有报价单相关的 JSON 数据。当用户再次提到合同数据时，LLM 可能直接复用对话历史中的模式生成 JSON。
2. **系统提示词对"必须调用工具"的约束不够强**：需要更明确的警告。

---

## 修复方案

在系统提示词中强化"必须调用工具"规则：

**文件**：`src/main/resources/prompts/sre-agent.md`

**修改**：在"响应原则"章节添加更明确的禁止行为

```markdown
## 偏好设置
用户偏好 `q` 模型（主要原因是 `qwen-turbo-latest` 在处理需要严格遵守工具调用规则的场景时可能表现不够稳定）。

**建议**：
1. 在系统提示词中添加更强烈的警告
2. 考虑在日志中添加诊断信息，3. 如果问题持续，可能需要在代码层面添加强制检查（检测输出是否为工具调用的结果）
```
