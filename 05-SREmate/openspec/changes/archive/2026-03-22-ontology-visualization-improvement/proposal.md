## Why

当前本体可视化页面（`ontology.html`）使用 vis-network 实现节点关系图，但节点和边的视觉效果较为朴素，缺乏视觉层次感和交互反馈，影响用户对本体结构的直观理解。

## What Changes

### 层级自适应（核心改动）
- **移除硬编码层级**：删除 `entityLevel` 手动映射，改为根据关系图自动计算层级
- **BFS 层级推导**：从根节点（Order）出发，通过广度优先遍历计算每个实体的层级深度
- **修复 ContractInstance 位置**：ContractInstance 将正确显示在最后一层（Order → Contract → ContractInstance）

### 节点视觉增强
- **渐变填充**：节点背景从纯色改为渐变色，增加立体感
- **多层阴影**：添加多层模糊阴影，突出节点层级
- **标签优化**：实体名加粗显示，displayName 以较小字号显示在下方
- **悬停效果**：鼠标悬停时节点发光、边框加粗
- **形状区分**：根据实体类型使用不同形状（核心实体用圆角矩形，普通实体用椭圆）

### 边视觉增强
- **渐变连接线**：从源节点颜色渐变到目标节点颜色
- **流动动画**：添加方向指示动画，直观展示关系方向
- **悬停高亮**：鼠标悬停时整条边高亮，标签放大
- **标签美化**：edge label 背景改为圆角气泡，配合领域配色

### 配色方案升级
- **提高对比度**：调整节点文字颜色确保可读性
- **领域配色优化**：使用更现代的配色方案，减少视觉疲劳
- **状态颜色**：选中/悬停/默认状态有明显区分

## Capabilities

### New Capabilities
- `ontology-graph-visuals`: 本体图谱节点和边的视觉样式规范

### Modified Capabilities
- 无（本次变更仅涉及前端样式，不影响后端规格）

## Impact

- **代码变更**：仅修改 `src/main/resources/static/ontology.html` 中的 CSS 样式和 vis-network 节点/边配置
- **无 API 变更**：不涉及后端接口
- **无依赖变更**：继续使用 vis-network 库
