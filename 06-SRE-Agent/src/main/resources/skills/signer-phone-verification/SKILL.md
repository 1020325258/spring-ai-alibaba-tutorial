# 签约人手机号验证

## 触发条件

用户说"XXX合同的签约人手机号与18303623350是否一致"、"C开头的合同的签约人手机号是XXX吗"、"核实合同签约人手机号"

## 排查步骤

### 第一步：查询合同签约人信息

从 Contract 出发，查询 ContractUser：

```
ontologyQuery(entity=Contract, value={合同号}, queryScope=ContractUser)
```

- 合同签约人属于合同维度，返回的 `records` 包含该合同下所有用户

### 第二步：筛选签约人并对比

从返回结果中：

1. **筛选签约人**：找到 `isSign = 1` 的记录（只有 1 条）
2. **获取签约人手机号**：读取 `phonePlain` 字段（明文手机号）
3. **对比结果**：将 `phonePlain` 与用户提供的手机号进行对比

## 字段说明

| 字段 | 含义 | 示例值 |
|------|------|--------|
| `isSign` | 是否为签约人：1-是，0-否 | 1 |
| `phonePlain` | 解密后的手机号（明文） | "18303623350" |
| `phone` | 加密后的手机号（密文） | 不用于对比 |
| `roleType` | 用户类型：1-业主 2-代理人 3-公司代办人 | 1 |

## 决策矩阵

| 签约人查询结果 | 对比结果 | 结论 |
|---------------|---------|------|
| 无 `isSign=1` 的记录 | - | 该合同无签约人信息，建议人工核实 |
| 有签约人，手机号一致 | ✅ 一致 | 签约人手机号与用户提供的一致 |
| 有签约人，手机号不一致 | ❌ 不一致 | 签约人手机号（phonePlain）与用户提供的不一致 |

## 输出格式

四段式自然语言分析结论：

**【数据查询】** 调用了哪些工具，查到了什么

**【分析】** 筛选出签约人（isSign=1），获取其 phonePlain 字段，与用户提供手机号对比

**【结论】** 手机号是否一致

**【建议】** 如不一致，建议下一步操作（如有）

## Example

用户: "C1775177384612302的签约人手机号与18303623350是否一致"

Action:
1. 查询合同 C1775177384612302 的签约人信息（ContractUser）
   - 返回多条用户记录，筛选 `isSign = 1` 的记录
   - 情形 A：无 `isSign=1` 的记录 → 结论：该合同无签约人信息，建议人工核实
   - 情形 B：有签约人记录 → 继续对比
2. 获取签约人的 `phonePlain` 字段，与用户提供的 `18303623350` 对比
   - 结果一：`phonePlain = "18303623350"` → 结论：一致 ✅
   - 结果二：`phonePlain = "13812345678"` → 结论：不一致 ❌，签约人手机号为 13812345678
